# MNV 對話範例

這個資料夾收 5 個**完全虛構**的對話範例，展示 MNV（密涅瓦思考專家）在不同情境下怎麼運作。

每個範例包含：情境設定 → 完整 MNV 對話流程 → 這個範例展示了什麼。

---

## 範例清單

| # | 主題 | 情境 | 主要展示 |
|---|------|------|----------|
| [01](./01-business-decision.md) | **商業策略卡關** | B2B SaaS 創辦人想轉 B2C | 診斷拆穿「跟風 = 解方」假設；誤用偵測 |
| [02](./02-life-choice.md) | **人生選擇** | 30 歲工程師接外派 offer | In-flight 修正抓到使用者跳開話題；🔀 替代方案 打開二選一 |
| [03](./03-product-design.md) | **產品設計卡關** | 獨立開發者考慮加新 feature | 「舉例 [角度名]」觸發詞流程；案例怎麼當錨點 |
| [04](./04-relationship.md) | **關係困惑** | 老同事三個月不回訊息 | 「檢查誤用 [角度名]」逐條對照；感性議題仍用結構思考 |
| [05](./05-self-doubt.md) | **自我懷疑 / 動機問題** | 35 歲主管「不知道為什麼還在做這個」 | 處理「沒有明確問題的問題」；不掉心理諮商陷阱 |

## 怎麼讀這些範例

**第一次讀**：建議從 01 開始按順序看，5 個範例累積展示 MNV 的完整工具箱。

**想看特定能力**：

- 想看 MNV **怎麼診斷** → 01、05
- 想看 MNV **怎麼用案例** → 03（觸發詞 `舉例 可證偽性`）
- 想看 MNV **怎麼自我修正** → 02（in-flight）
- 想看 MNV **怎麼偵測誤用** → 01（順帶提醒）、04（觸發詞 `檢查誤用 假設`）
- 想看 MNV **在感性議題上怎麼運作** → 04、05

---

## MNV 的核心紀律（從這 5 個範例看出來）

1. **不給答案，給下一個該想的問題**
2. **不安慰、不評價、不堆術語**
3. **2-4 個角度就夠**，不貪多
4. **走錯及時換**比走完才檢討好十倍（in-flight 修正）
5. **誤用一次只指 1 個**，溫和語氣，不開講座

完整規則見 repo 根目錄的 [SKILL.md](../SKILL.md)。
